# 🌲 TrailQuest — Project Plan

> Turn **any trail into an AI-powered scavenger hunt game**. Paste an AllTrails link,
> and OpenAI generates a themed hunt; point a camera at things and AI **vision** verifies
> your finds and awards points.

Built at **RampHacks**.

---

## 1. The Idea

Every year millions of people hike the same trails the same way. TrailQuest turns any
trail on Earth into a personalized adventure game — **generated by AI, verified by AI** —
from a single link. It also runs in **Venue Mode** so judges can play live indoors with a
laptop webcam.

**One-liner (pitch opener):**
> "We turn any trail into a personalized adventure game — generated by AI, verified by AI —
> in one link."

## 3. How It Works (User Flow)

1. **Input** — paste an AllTrails link / trail name, or pick **Venue Mode** for indoors.
2. **Generate** — OpenAI creates a themed 6-challenge hunt tailored to the place.
3. **Play** — each challenge asks you to photograph something ("find peeling bark",
   "photograph water moving over rock").
4. **Verify** — snap a webcam photo → OpenAI **vision** decides if it matches → points.
5. **Finish** — AI writes a celebratory **trail-journal recap** of the adventure.

## 4. Architecture

```
Browser (static/index.html)
  ├─ Webcam capture (getUserMedia + canvas → base64 JPEG)
  ├─ Game UI: modes, challenge cards, scoring, recap
  └─ fetch() → Flask API
        │
Flask backend (app.py)
  ├─ POST /api/generate  → GPT-4o → themed hunt JSON
  ├─ POST /api/verify    → GPT-4o vision → {match, confidence, reason}
  └─ POST /api/recap     → GPT-4o → trail-journal text
        │
     OpenAI API (gpt-4o: generation + vision)
```

**Stack:** Flask + plain HTML/JS (no build step), OpenAI `gpt-4o`. Runs entirely on the
demo laptop's webcam — **no iPhone required**.

**Key design choices:**
- **No GPS** — removed for an indoor, venue-only build; pacing is by challenge order.
- **AllTrails link = UX sugar** — the AI generates from the trail *name*, so there is no
  scraping / no-API risk.
- **Lazy OpenAI client** — server boots and serves the page even without a key; AI calls
  return a clean JSON error.
- **Lenient verification** — the vision judge is prompted to be generous so photos accept
  smoothly on stage.

## 6. Run It (venue laptop, ~1 min)

```bash
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env        # paste your OpenAI key into .env
python app.py
```
Open **http://localhost:5001** in Chrome and allow camera access.

## 7. Demo Script (all indoor, venue-safe)

1. **Venue Mode (live, ~60s)** — hand a judge the laptop → they photograph a nearby object
   → AI verifies live → points fire. *This is the Audience-Favorite hook.*
2. **Trail Mode (the story)** — paste an AllTrails link → a full themed nature hunt
   generates in ~10s on screen. *Sells Best Use of OpenAI.*
3. **Finish** — complete a hunt → show the AI-written trail-journal recap.

**Safety net:** keep a hunt pre-generated in case wifi is flaky; verification is tuned
lenient so photos accept on stage.

## 9. Risks & Mitigations

| Risk | Mitigation |
| --- | --- |
| No public AllTrails API | Generate from the trail *name*; link is optional sugar. |
| Wifi / rate limits on stage | Pre-generate a hunt; keep the page open. |
| Vision false-negatives | Prompt the judge model to be lenient; accept confidence > 0.6. |
| Camera permission issues | Test in Chrome first; clear allow-camera prompt on load. |
